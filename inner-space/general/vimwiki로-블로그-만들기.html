vimwiki 사용하기 ​

kickstart.nvim으로 간단하게 플러그인 관리를 하고 있으므로, init.lua에 다음과 같은 부분을 추가한다. 아래는 vimwiki를 설치하는 부분이다.

lua
vim.g.vimwiki_list = {
  {
    path = '~/kepp-wiki/',
    syntax = 'markdown',
    ext = 'md',
    diary_rel_path = 'diary/',
  },
}
vim.g.vimwiki_global_ext = 0

vim.pack.add { gh 'vimwiki/vimwiki' }

nvim을 실행해 vimwiki가 설치되면, <leader>ww로 메인 인덱스를 열 수 있다. 여기서는 ~/kepp-wiki/로 설정해두었다. 이렇게 설정해두면 ~/kepp-wiki/index.md를 자동으로 열어준다. 다이어리 인덱스는 <leader>wi로 열고, 오늘의 다이어리 파일은 <leader>w<leader>w로 열 수 있다. 할 일 관리는 :VimwikiToggleList로 한다. 현재 리스트 checkbox로 바꾸거나 할 일을 체크할 수 있다.

Github Actions로 블로그 발행하기 ​

이제 만든 vimwiki를 블로그로 발행해보자. 이하의 내용은 기본적인 git 사용법을 안다는 전제하에 이루어진다.

우선 블로그를 발행하기 위해서는 다음과 같은 조건이 필요했다.

  1. git으로 간단하게 발행이 가능할 것
  2. 비용이 들지 않을 것
  3. push시에 자동으로 블로그 업데이트가 이루어질 것

해당 조건을 모두 만족하는 것은 github actions였다. 특정 action (여기서는 push)이 발생할 경우 자동으로 설정된 동작을 수행할 수 있다는 점이 컸다. 또한 이미 있는 repo에 대해 빌드를 하고, 외부 repo에 이를 다시 push하는 기능을 구현할 수 있어 블로그 배포에 안성맞춤이었다.

다만 github actions만으로는 부족하고, 이미 있는 md 문서들을 블로그에 맞는 html로 변환시켜주는 툴이 필요하다. 그러기 위한 툴로는 VitePress, Quartz, Astro, Hugo가 있었는데, 최종적으로는 VitePress를 선택했다. 이는 현재 vimwiki 구조에 가장 잘 맞기 때문이었다. 다른 툴들은 Obsidian에 더 적합하거나, 다른 구성해야 할 것이 많아 사용법이 복잡했다.

또한 특정 폴더만을 지정해 빌드할 수 있어서 더욱 마음에 들었다.

VitePress 설정 ​

VitePress를 사용하기 위해서는 npm scripts를 이용해야 한다. 따라서 kepp-wiki/package.json에 다음과 같은 스크립트를 추가한다.

json
"scripts": {
    "recent": "node scripts/update-recent-pages.mjs",
    "docs:dev": "npm run recent && vitepress dev blog",
    "docs:build": "npm run recent && vitepress build blog",
    "docs:preview": "vitepress preview blog"
}
  • recent는 임의로 추가한 스크립트로, 최근 수정된 블로그 글 목록을 출력하기 위해 만든 스크립트이다.
  • docs:dev는 로컬 개발 서버를 실행하는 것으로, 테스트를 위한 스크립트이다.
  • build는 정적 사이트를 실제로 빌드하는 스크립트이다.
  • preview는 빌드 결과를 미리보기하는 스크립트이다.

이후 kepp-wiki/blog/.vitepress.config.mts에 다음과 같은 내용을 넣는다. 대충 kepp-wiki에 있는 blog 폴더만을 지정해 빌드하고, outta-space와 inner-space의 기본 페이지를 지정해주는 작업이다.

mts
export default defineConfig({
  title: "Kepp Blog",
  description: "Personal blog",
  base: "/",
  cleanUrls: true,
  themeConfig: {
    nav: [
      { text: "Home", link: "/" },
      { text: "Outta Space", link: "/outta-space/" },
      { text: "Inner Space", link: "/inner-space/general/" }
    ]
  }
});

github actions 추가하기 ​

이제 github actions가 정상적으로 동작할 수 있도록 다음과 같은 코드를 kepp-wiki/.github/workflow/publich.yml에 추가한다.

yml
name: Publish Blog

  on:
    push:
      branches: [main]
      paths:
        - "blog/**"
        - "scripts/**"
        - "package.json"
        - "package-lock.json"
        - ".github/workflows/publish.yml"
    workflow_dispatch:

  jobs:
    publish:
      runs-on: ubuntu-latest

      steps:
        - name: Checkout private source
          uses: actions/checkout@v4
          with:
            fetch-depth: 0

        - name: Setup Node
          uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: npm

        - name: Install dependencies
          run: npm ci

        - name: Build VitePress site
          run: npm run docs:build

        - name: Publish to public repo
          uses: peaceiris/actions-gh-pages@v4
          with:
            personal_token: ${{ secrets.BLOG_DEPLOY_TOKEN }}
            external_repository: keplerisgone/keplerisgone.github.io
            publish_branch: main
            publish_dir: blog/.vitepress/dist

위 스크립트는 대충 main 브랜치에서 push가 일어나고, 블로그에 변화가 생겼을 경우 자동으로 빌드를 해주는 action을 나타낸 것이다. ubuntu 가상 머신에서 npm를 설치하고, npm scripts를 실행하는 과정을 담고 있다.

이러면 블로그 발행 끝! 사실 글에 담지 못한 github key 발행 과정, 최근 페이지 띄우기 기능 등이 많지만 이는 다른 블로그 글로 다시 돌아오도록 하겠다.
