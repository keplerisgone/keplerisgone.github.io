운영체제 ​

SJF와 SRT ​

  • 유형: CPU 스케줄링 알고리즘의 특징을 보고 명칭을 구분하는 문제
  • 규칙: SJF는 실행 시간이 짧은 작업을 우선하고, SRT는 남은 실행 시간이 더 짧은 작업이 도착하면 CPU를 선점한다.
  • 함정: SJF의 선점형 형태가 SRT이며, 새 프로세스의 실행 시간과 현재 프로세스의 남은 시간을 비교한다.
  • 예시: 현재 프로세스의 남은 시간이 5초이고 3초짜리 프로세스가 도착하면 SRT에서는 새 프로세스가 선점한다.
  • 메모: SJF는 짧은 Job, SRT는 짧은 Remaining Time으로 기억한다.

라운드로빈 평균 대기시간 ​

  • 유형: 라운드로빈 스케줄링의 실행 순서를 구한 뒤 평균 대기시간을 계산하는 문제
  • 규칙: 대기시간 = 완료시간 - 도착시간 - 실행시간이며, 평균 대기시간은 각 프로세스 대기시간의 합을 프로세스 수로 나눈다.
  • 함정: CPU 실행이 끝난 시간을 그대로 대기시간으로 사용하지 말고 도착시간과 실제 실행시간을 빼야 한다.
  • 예시: P2의 대기시간은 완료시간 8 - 도착시간 1 - 실행시간 4 = 3ms이다.
  • 메모: 반환시간 = 완료시간 - 도착시간, 대기시간 = 반환시간 - 실행시간, 응답시간 = 최초 실행시간 - 도착시간

Java 값 전달과 참조값 복사 ​

  • 유형: Java의 값 전달 방식과 배열·문자열의 변경 결과를 묻는 문제
  • 규칙: Java는 항상 Call by Value이며, 배열은 참조값이 복사되어 전달되고 String 변수도 참조값 자체가 복사되어 전달된다.
  • 함정: data[0] 변경은 원본 배열에 반영되지만 매개변수 s에 새 문자열을 대입해도 main의 s는 바뀌지 않는다.
  • 예시: data[0] = s로 배열의 "A"가 "B"로 바뀌고, s = "Z"는 change 메서드 내부에서만 적용된다.
  • 메모: Java에 Call by Reference는 없다고 기억한다.

Java 람다와 예외 처리 ​

java
public class Main {
    static interface F {
        int apply(int x) throws Exception;
    }

    public static int run(F f) {
        try {
            return f.apply(3);
        } catch (Exception e) {
            return 7;
        }
    }

    public static void main(String[] args) {
        F f = (x) -> {
            if (x > 2) {
                throw new Exception();
            }
            return x * 2;
        };

        System.out.print(run(f) + run((int n) -> n + 9));
    }
}
  • 유형: 함수형 인터페이스, 람다식, 예외 처리의 실행 결과
  • 규칙: run은 람다에 3을 전달하고 정상 실행 시 반환값을, 예외 발생 시 catch의 7을 반환한다.
  • 함정: 첫 번째 람다는 3 > 2이므로 예외가 발생하지만, 두 번째 람다는 예외 없이 3 + 9를 반환한다.
  • 예시: run(f) = 7, run((int n) -> n + 9) = 12이므로 합계는 19다.
  • 메모: throws Exception은 예외를 던질 수도 있다는 뜻이다.

Java 오버라이딩과 static 메서드 숨김 ​

java
public class Main {
    public static class Parent {
        public int x(int i) {
            return i + 2;
        }

        public static String id() {
            return "P";
        }
    }

    public static class Child extends Parent {
        public int x(int i) {
            return i + 3;
        }

        public String x(String s) {
            return s + "R";
        }

        public static String id() {
            return "C";
        }
    }

    public static void main(String[] args) {
        Parent ref = new Child();
        System.out.println(ref.x(2) + ref.id());
    }
}
  • 유형: Java의 오버라이딩과 static 메서드 숨김을 구분하는 문제
  • 규칙: 인스턴스 메서드는 실제 객체 타입을 따르고, static 메서드는 참조 변수의 선언 타입을 따른다.
  • 함정: Child의 static id()는 오버라이딩이 아니라 Parent의 id()를 숨기는 메서드다.
  • 예시: Parent ref = new Child()에서 ref.x(2)는 Child.x(), ref.id()는 Parent.id()가 호출된다.
  • 메모: static 메서드는 동적 바인딩되지 않으므로 참조 타입 Parent 기준으로 P가 선택된다.

Java 객체 참조 배열 ​

java
public class Main {
    public static class BO {
        public int v;

        public BO(int v) {
            this.v = v;
        }
    }

    public static void main(String[] args) {
        BO a = new BO(1);
        BO b = new BO(2);
        BO c = new BO(3);

        BO[] arr = {a, b, c};

        BO t = arr[0];
        arr[0] = arr[2];
        arr[2] = t;

        arr[1].v = arr[0].v;

        System.out.println(a.v + "a" + b.v + "b" + c.v);
    }
}
  • 유형: 객체 참조 배열의 요소 교환과 객체 필드 변경 추적
  • 규칙: 배열 요소를 교환하면 객체 내부 값이 아니라 배열에 저장된 참조가 교환된다.
  • 함정: arr[0]과 arr[2]를 바꿔도 변수 a와 c가 가리키는 객체 자체는 바뀌지 않는다.
  • 예시: 교환 후 arr은 [c, b, a]가 되고 arr[1].v = arr[0].v로 b.v만 3이 된다.
  • 메모: 배열에서 객체를 교환한 것은 a, c의 값 교환이 아니라 참조 위치 교환이다.

Python 리스트, 딕셔너리, 집합 ​

python
lst = [1, 2, 3]
dst = {i: i * 2 for i in lst}
s = set(dst.values())

lst[0] = 99
dst[2] = 7
s.add(99)

print(len(s & set(dst.values())))
  • 유형: Python의 리스트·딕셔너리·집합 생성과 교집합 연산 추적
  • 규칙: 리스트는 순서 있는 값 모음, 딕셔너리는 키와 값의 대응, 집합은 중복 없는 값 모음이다.
  • 함정: dst와 s는 생성 당시 값을 복사해 구성하므로 이후 lst나 dst를 변경해도 자동으로 함께 바뀌지 않는다.
  • 예시: s & set(dst.values())는 두 집합에 공통으로 존재하는 값만 남긴다.
  • 메모: list, dictionary의 {}, set의 차이와 각 자료구조가 독립적으로 생성된다는 점을 구분한다.

C 문자열 포인터와 연결 리스트 ​

c
#include <stdio.h>
#include <stdlib.h>

struct node {
    char c;
    struct node* p;
};

struct node* func(char* s) {
    struct node* h = NULL, *n;

    while (*s) {
        n = malloc(sizeof(struct node));
        n->c = *s++;
        n->p = h;
        h = n;
    }

    return h;
}

int main() {
    struct node* n = func("BEST");

    while (n) {
        putchar(n->c);
        struct node* t = n;
        n = n->p;
        free(t);
    }

    return 0;
}
  • 유형: C 문자열 포인터와 연결 리스트의 삽입·순회 결과 추적
  • 규칙: char* s는 문자열을 한 글자씩 가리키고, node.c는 현재 글자 하나를 저장하며, 새 노드는 항상 맨 앞에 연결한다.
  • 함정: s와 c는 같은 종류의 변수가 아니다. s는 문자열 위치를 가리키는 포인터이고, c는 문자 한 개를 저장하는 필드다.
  • 예시: "BEST"를 B→E→S→T 순서로 읽지만 앞쪽에 삽입하므로 리스트는 T→S→E→B가 된다.
  • 메모: 포인터는 여러 문자의 위치, char는 문자 하나다.
