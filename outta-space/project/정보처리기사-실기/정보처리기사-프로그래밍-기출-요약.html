Java ​

조건식과 논리 연산자 ​

  • 유형: Java 조건문, 논리 연산자, 비교 연산자, 연산자 우선순위
  • 규칙: Java 조건식은 ! → 비교 연산자(==, >, !=) → & → ^ → | 순서로 계산된다.
  • 함정: ^는 논리 XOR이므로 양쪽 조건의 참·거짓이 서로 다를 때만 true가 된다. &와 |는 단축 평가 없이 양쪽 조건을 모두 계산한다.
  • 예시: 첫 if가 참이면 w = x + y = 7이 되고, 내부 if도 참이면 System.out.println(w)로 7이 출력된다.

Java 연산자 우선순위 ​

  • 유형: 조건식 계산 순서
  • 규칙: 괄호 → 단항 ! → 산술 연산자 → 비교 연산자(==, !=, >, <) → 논리 AND & → 논리 XOR ^ → 논리 OR | 순서로 계산한다.
  • 함정: ^는 제곱이 아니라 XOR이다. 조건식에서는 두 피연산자의 결과가 다를 때만 true가 된다.
  • 예시: false ^ true는 true, true ^ true는 false다.

10진수에서 2진수 변환 ​

  • 유형: Java 반복문, 10진수에서 2진수 변환, 나머지 연산
  • 규칙: n > 0인 동안 n % 2를 배열에 저장하고 n /= 2로 몫을 줄이면 2진수의 비트가 역순으로 저장된다.
  • 함정: 배열에는 낮은 자리 비트부터 저장되므로 출력할 때는 뒤에서부터 읽어야 한다.
  • 예시: n = 10이면 나머지 0, 1, 0, 1이 저장되고, 배열을 뒤에서부터 출력하면 00001010이 된다.

while, continue, 누적합 ​

  • 유형: Java while문, continue, 나머지 연산, 누적합
  • 규칙: a++ 후 a % 2 == 1이면 continue로 sum += a를 건너뛰므로 짝수만 누적된다.
  • 함정: continue는 반복문 전체를 종료하지 않는다. 현재 반복의 남은 코드만 건너뛰고 다음 반복으로 넘어간다.
  • 예시: 2 + 4 + 6 + 8 + 10 = 30
  • 메모: 침착하게 풀자.

static 메서드 ​

  • 유형: Java static 메서드, 클래스명 호출, 삼항 연산자
  • 규칙: 클래스명.메서드명() 형태로 호출하려면 해당 메서드는 static으로 선언되어야 한다.
  • 함정: static이 없으면 check()는 인스턴스 메서드가 되어 new Test().check(1)처럼 객체를 생성해야 호출할 수 있다.
  • 예시: Test.check(1)을 쓰려면 static String check(int num)이 필요하고, 1 >= 0이므로 "positive"가 반환된다.

생성자, this, 필드 변경 ​

  • 유형: Java 클래스, 생성자, this, 객체 필드값 변경, 반복문 누적 계산
  • 규칙: 생성자 cond(3)으로 처음 a = 3이 되지만, 이후 obj.a = 5로 객체의 필드 a가 5로 변경된다. func() 안에서는 변경된 a = 5를 기준으로 반복문을 수행한다.
  • 함정: 생성자에서 넣은 3을 계속 쓰는 것이 아니라, obj.a = 5 이후에는 객체 내부 a가 5다. 또한 func()의 반환값은 b가 아니라 a + b다.
  • 예시: b = 1에서 시작, i = 1~4 동안 b += 5 * i → b = 51, func()는 a + b = 56 반환, main에서 obj.a + b = 61
  • 메모: 숫자 계산 문제는 머리로 하지 말고 표로 풀자.

오버라이딩과 super ​

  • 유형: Java 상속, 메서드 오버라이딩, super 호출
  • 규칙: 자식 클래스가 부모 클래스와 같은 이름·매개변수의 메서드를 다시 정의하면 오버라이딩이며, 자식 객체에서 호출하면 자식 메서드가 우선 실행된다.
  • 함정: super.sun(x, y)는 자식 클래스 안에서 부모 클래스의 sun(x, y)를 호출한다는 뜻이다.
  • 예시: a1.sun(3,2) = 3 + 2 = 5, a2.sun(3,2) = 3 - 2 + super.sun(3,2) = 1 + 5 = 6, 출력은 5 + 6 = 11

상속 생성자와 문자열 연결 ​

  • 유형: Java 상속, 생성자, super(), super.메서드(), 문자열 연결
  • 규칙: 자식 클래스 생성자에서 super(a)를 호출하면 부모 클래스 생성자가 먼저 실행되어 this.a = a가 수행된다. 이후 super.display()는 부모 클래스의 display() 메서드를 호출한다.
  • 함정: System.out.println("a=" + a)에서 +는 숫자 덧셈이 아니라 문자열 연결 연산으로 동작한다.
  • 예시: new B(10) → super(10)으로 A 생성자 실행 → A의 필드 a = 10 저장 → super.display() 호출 → a=10

Runnable과 Thread ​

  • 유형: Java 인터페이스, Runnable, Thread 생성, run() / start()
  • 규칙: Runnable을 구현한 클래스는 반드시 run() 메서드를 구현해야 하며, new Thread(new Car())처럼 Runnable 객체를 Thread 생성자에 전달할 수 있다.
  • 함정: run()을 직접 호출하는 것이 아니라 t1.start()를 호출해야 새 스레드가 시작되고, 그 결과 내부적으로 run()이 실행된다.
  • 예시: Thread t1 = new Thread(new Car()); → t1.start(); → Car의 run() 실행

static 변수와 싱글턴 ​

  • 유형: Java static 변수, 싱글턴 패턴, 객체 참조 공유
  • 규칙: static 변수는 객체마다 따로 생기지 않고 클래스에 하나만 존재한다. _inst가 static이므로 Connection.get()을 여러 번 호출해도 같은 Connection 객체를 반환한다.
  • 함정: conn1, conn2, conn3은 서로 다른 변수처럼 보이지만 실제로는 모두 같은 Connection 객체를 가리킨다.
  • 예시: 같은 객체의 count()를 세 번 호출하면 conn1.getCount() = 3

업캐스팅과 동적 바인딩 ​

  • 유형: Java 상속, 업캐스팅, 오버라이딩, 동적 바인딩
  • 규칙: 자식 객체는 부모 타입 참조 변수에 담을 수 있다. Parent pa = new Child();는 Child 객체를 Parent 타입으로 참조하는 업캐스팅이다.
  • 함정: 변수 타입은 Parent지만 실제 객체는 Child이므로, 오버라이딩된 메서드 show()를 호출하면 Child의 show()가 실행된다.
  • 예시: Parent pa = new Child(); pa.show(); → Child의 show() 실행 → child 출력

추상 클래스, 오버라이딩, 오버로딩 ​

  • 유형: Java 추상 클래스, 상속, 오버라이딩, 오버로딩, 필드 숨김
  • 규칙: 오버라이딩은 메서드 이름과 매개변수 형태가 같아야 한다. Car는 getName(String)을 구현했지만 getName()은 구현하지 않았으므로 Vehicle의 getName()이 실행된다.
  • 함정: Vehicle obj = new Car("Spark")라서 무조건 Car 메서드가 실행되는 것이 아니다. 호출한 메서드 시그니처가 getName()이면 Vehicle에 있는 getName()이 호출된다.
  • 예시: obj.getName() → Vehicle.getName() 실행 → "Vehicle name : " + Vehicle의 name 출력 → Vehicle name : Spark

Python ​

range와 비트 시프트 ​

  • 유형: Python 반복문, range, 비트 시프트 연산자, 변수 덮어쓰기
  • 규칙: range(1, 3)은 1, 2까지만 반복하고, a >> i는 a를 오른쪽으로 i비트 시프트한다.
  • 함정: result = a >> i는 반복마다 값을 누적하지 않고 덮어쓴다. 따라서 마지막 반복의 결과가 최종 출력에 남는다.
  • 예시: i = 1이면 100 >> 1 = 50 → result = 51, i = 2이면 100 >> 2 = 25 → result = 26, 최종 출력은 26이다.

2차원 리스트와 출력 형식 ​

  • 유형: Python 2차원 리스트, 인덱싱, 중첩 반복문, print(end=' ')
  • 규칙: 리스트 인덱스는 0부터 시작하며, lol[2][1]은 세 번째 리스트의 두 번째 값을 의미한다.
  • 함정: print(item, end=' ')는 줄바꿈 없이 공백을 붙여 출력하고, 안쪽 반복문이 끝난 뒤 print()가 실행되어 다음 줄로 넘어간다.
  • 예시: print(lol[0])은 [1, 2, 3], print(lol[2][1])은 7, 중첩 반복문 출력은 1 2 3 / 4 5 / 6 7 8 9 형태가 된다.

문자열 슬라이싱과 포매팅 ​

  • 유형: Python 문자열 슬라이싱, 문자열 연결, 문자열 포매팅
  • 규칙: 문자열[시작:끝]은 시작 인덱스부터 끝 인덱스 직전까지 자른다. 문자열 안의 %s는 뒤에 오는 값을 문자열로 끼워 넣는 자리다.
  • 함정: "R AND %s" % "STR"에서 %는 나머지 연산이 아니라 문자열 포매팅이다.
  • 예시: a[0:3] = "REM", a[12:16] = "EMBE" → b = "REMEMBE", c = "R AND STR" → b + c = "REMEMBER AND STR"

set ​

  • 유형: Python set, add(), remove(), update(), 중복 제거
  • 규칙: set은 중복을 허용하지 않고, 순서를 보장하지 않는 자료형이다. add()는 원소 하나를 추가하고, remove()는 원소 하나를 삭제하며, update()는 여러 원소를 한 번에 추가한다.
  • 함정: {'한국', '중국', '일본'}은 튜플이 아니라 set이다. 이미 있는 값은 추가해도 변화가 없다.
  • 예시: {'한국','중국','일본'} → 베트남 추가 → 중국 추가 무시 → 일본 삭제 → 한국/홍콩/태국 update → {'한국','중국','베트남','홍콩','태국'}

map과 lambda ​

  • 유형: Python map, lambda, list 변환
  • 규칙: map(함수, 반복자료)는 반복자료의 각 원소에 함수를 하나씩 적용한다. lambda num: num + 100은 입력값 num에 100을 더해 반환하는 익명 함수다.
  • 함정: map의 결과는 바로 리스트처럼 출력되지 않으므로 list()로 감싸 리스트로 변환한다.
  • 예시: [1,2,3,4,5]의 각 원소에 +100 적용 → [101,102,103,104,105]

C ​

문자열 포인터와 길이 계산 ​

  • 유형: C언어 문자열 포인터, 널 문자('\0'), 사용자 정의 길이 함수
  • 규칙: 문자열은 끝에 널 문자 '\0'이 있으며, while (*p != '\0') 동안 p++와 r++을 반복하면 문자열 길이를 구할 수 있다.
  • 함정: 마지막 널 문자 '\0'은 문자열 길이에 포함하지 않는다.
  • 예시: len("2022") = 4, len("202207") = 6이므로 a + b = 10이다.

문자열 포인터 산술과 출력 ​

  • 유형: C언어 문자열 포인터, 포인터 산술, 역참조, 문자·문자열 출력
  • 규칙: p는 문자열 "KOREA"의 첫 글자 주소를 가리키고, p + n은 n번째 문자 주소로 이동하며, *(p + n)은 그 위치의 문자 값을 꺼낸다.
  • 함정: %s는 해당 주소부터 '\0' 전까지 문자열을 출력하고, %c는 포인터가 가리키는 문자 하나만 출력한다.
  • 예시: p는 "KOREA", p + 3은 "EA", *p는 'K', *(p + 3)은 'E', *p + 2는 'K'의 ASCII 값에 2를 더한 'M'이다.

구조체 배열 초기화 ​

  • 유형: C언어 구조체 배열, 구조체 초기화, 구조체 포인터, -> 연산자
  • 규칙: struct 초기화 리스트는 구조체 내부에 선언된 멤버 순서대로 값이 들어가며, 생략된 int 멤버는 0으로 초기화된다.
  • 함정: {"데이터1", 95, 88}은 nae = "데이터1", os = 95, db = 88까지만 채우고 hab, hhab은 0으로 남는다.
  • 예시: struct jsu { char nae[12]; int os, db, hab, hhab; };에서 {"데이터2", 84, 91} → nae = "데이터2", os = 84, db = 91, hab = 0, hhab = 0

소수 판별과 최대 소인수 ​

  • 유형: C언어 함수, 소수 판별, 약수 검사, 최댓값 갱신
  • 규칙: i가 소수이고 number를 나누어떨어지게 하면 max_div = i로 갱신하므로, 최종적으로 number의 가장 큰 소인수가 출력된다.
  • 함정: isPrime(i)는 i가 소수인지 검사하고, number % i == 0은 i가 number의 약수인지 검사한다. 반복문이 작은 수부터 큰 수까지 돌기 때문에 마지막에 저장된 소인수가 가장 큰 소인수다.
  • 예시: 13195 = 5 × 7 × 13 × 29이므로 가장 큰 소인수는 29

재귀와 팩토리얼 ​

  • 유형: C언어 재귀 함수, 종료 조건, 팩토리얼 계산
  • 규칙: 함수가 자기 자신을 다시 호출하면 재귀 함수이며, a <= 1이면 1을 반환하고 그 외에는 a * func(a - 1)을 반환한다.
  • 함정: 재귀 함수는 반드시 종료 조건이 있어야 하며, 여기서는 if (a <= 1) return 1;이 종료 조건이다.
  • 예시: func(5) = 5 * func(4) = 5 * 4 * 3 * 2 * 1 = 120
