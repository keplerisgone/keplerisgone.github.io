[프로그래밍] 유형: Java 조건문, 논리 연산자, 비교 연산자, 연산자 우선순위 규칙: Java 연산자 우선순위는 ! → 비교연산자(==, >, !=) → & → ^ → | 순서로 계산된다. 함정: ^는 논리 XOR이므로 양쪽 조건이 서로 다를 때만 true이고, &와 |는 단축 평가 없이 양쪽 조건을 모두 계산한다. 예시: 첫 if는 참이므로 w = x + y = 7, 내부 if도 참이 되어 System.out.println(w)로 7 출력

[Java 연산자 우선순위] 유형: 조건식 계산 순서 규칙: 괄호 → 단항 ! → 산술 → 비교 ==, !=, >, < → 논리 AND & → 논리 XOR ^ → 논리 OR | 순서로 계산한다. 함정: ^는 제곱이 아니라 XOR이며, 조건식에서는 양쪽 결과가 다를 때 true가 된다. 예시: false ^ true = true, true ^ true = false

[프로그래밍] 유형: Python 반복문, range, 비트 시프트 연산자, 변수 덮어쓰기 규칙: range(1, 3)은 1, 2까지만 반복하고, a >> i는 a를 오른쪽으로 i비트 시프트하여 2^i로 나눈 몫과 같다. 함정: result = a >> i가 반복마다 기존 result를 누적하지 않고 새 값으로 덮어쓰므로, 마지막 반복 i=2의 결과만 최종 출력에 남는다. 예시: i=1 → 100 >> 1 = 50 → result=51, i=2 → 100 >> 2 = 25 → result=26, 최종 출력 26

[프로그래밍] 유형: Java 클래스, 생성자, this, 객체 필드값 변경, 반복문 누적 계산 규칙: 생성자 cond(3)으로 처음 a=3이 되지만, 이후 obj.a = 5로 객체의 필드 a가 5로 변경된다. func() 안에서는 이 변경된 a=5를 기준으로 반복문을 수행한다. 함정: 생성자에서 넣은 3을 계속 쓰는 것이 아니라, obj.a = 5 이후에는 객체 내부 a가 5이다. 또한 func()의 반환값은 b가 아니라 a + b이다. 예시: b=1에서 시작, i=1~4 동안 b += 5*i → b=1+5+10+15+20=51, func()는 a+b=5+51=56 반환, main에서 obj.a+b = 5+56 = 61 메모: 숫자 계산 문제는 머리로 하지 말고 표로 풀자.

[프로그래밍] 유형: Java 상속, 메서드 오버라이딩, super 호출 규칙: 자식 클래스가 부모 클래스와 같은 이름·매개변수의 메서드를 다시 정의하면 오버라이딩이며, 자식 객체에서 호출하면 자식 메서드가 우선 실행된다. 함정: super.sun(x, y)는 자식 클래스 안에서 부모 클래스의 sun(x, y)를 호출한다는 뜻이다. 따라서 a2.sun(3,2)는 자식의 sun을 실행하지만, 그 내부의 super.sun(3,2)는 부모의 sun을 실행한다. 예시: a1.sun(3,2)=3+2=5, a2.sun(3,2)=3-2+super.sun(3,2)=1+5=6, 출력은 5+6=11

[프로그래밍] 유형: Java 상속, 생성자, super(), super.메서드(), 문자열 연결 규칙: 자식 클래스 생성자에서 super(a)를 호출하면 부모 클래스 A의 생성자가 먼저 실행되어 this.a = a가 수행된다. 이후 super.display()는 부모 클래스의 display() 메서드를 호출한다. 함정: System.out.println("a=" + a)에서 +는 숫자 덧셈이 아니라 문자열 연결 연산으로 동작한다. 문자열 "a="와 정수 a가 만나면 정수 a가 문자열로 변환되어 "a=10"이 출력된다. 예시: new B(10) → B 생성자 실행 → super(10)으로 A 생성자 실행 → A의 필드 a=10 저장 → super.display() 호출 → "a=" + 10 출력 → a=10

[프로그래밍] 유형: Java 인터페이스, Runnable, Thread 생성, run() / start() 규칙: Runnable을 구현한 클래스는 반드시 run() 메서드를 구현해야 하며, new Thread(new Car())처럼 Runnable 객체를 Thread 생성자에 전달할 수 있다. 함정: run()을 직접 호출하는 것이 아니라 t1.start()를 호출해야 새 스레드가 시작되고, 그 결과 내부적으로 run()이 실행된다. 예시: Thread t1 = new Thread(new Car()); → t1.start(); → Car의 run() 실행

[프로그래밍] 유형: Java static 변수, 싱글턴 패턴, 객체 참조 공유 규칙: static 변수는 객체마다 따로 생기지 않고 클래스에 하나만 존재한다. _inst가 static이므로 Connection.get()을 여러 번 호출해도 같은 Connection 객체를 반환한다. 함정: conn1, conn2, conn3은 서로 다른 변수처럼 보이지만 실제로는 모두 같은 Connection 객체를 가리킨다. 그래서 count()를 세 번 호출하면 같은 객체의 count가 3까지 증가한다. 예시: conn1 = Connection.get(), conn2 = Connection.get(), conn3 = Connection.get() 모두 같은 객체 → count 세 번 증가 → conn1.getCount() = 3

[프로그래밍] 유형: Java 상속, 업캐스팅, 오버라이딩, 동적 바인딩 규칙: 자식 객체는 부모 타입 참조 변수에 담을 수 있다. Parent pa = new Child();는 Child 객체를 Parent 타입으로 참조하는 업캐스팅이다. 함정: 변수 타입은 Parent지만 실제 객체는 Child이므로, 오버라이딩된 메서드 show()를 호출하면 Child의 show()가 실행된다. 예시: Parent pa = new Child(); pa.show(); → Child의 show() 실행 → child 출력

[프로그래밍] 유형: Java 추상 클래스, 상속, 오버라이딩, 오버로딩, 필드 숨김 규칙: 오버라이딩은 메서드 이름과 매개변수 형태가 같아야 한다. Car는 getName(String)을 구현했지만 getName()은 구현하지 않았으므로 Vehicle의 getName()이 실행된다. 함정: Vehicle obj = new Car("Spark")라서 무조건 Car 메서드가 실행되는 것이 아니다. 호출한 메서드 시그니처가 getName()이면 Vehicle에 있는 getName()이 호출된다. 예시: obj.getName() → Vehicle.getName() 실행 → "Vehicle name : " + Vehicle의 name 출력 → Vehicle name : Spark

[프로그래밍] 유형: Python 문자열 슬라이싱, 문자열 연결, 문자열 포매팅 규칙: 문자열[시작:끝]은 시작 인덱스부터 끝 인덱스 직전까지 자른다. 문자열 안의 %s는 뒤에 오는 값을 문자열로 끼워 넣는 자리이다. 함정: "R AND %s" % "STR"에서 %는 나머지 연산이 아니라 문자열 포매팅이다. %s 자리에 "STR"이 들어가서 "R AND STR"이 된다. 예시: a[0:3]="REM", a[12:16]="EMBE" → b="REMEMBE", c="R AND STR" → b+c="REMEMBER AND STR"

[프로그래밍] 유형: Python set, add(), remove(), update(), 중복 제거 규칙: set은 중복을 허용하지 않고, 순서를 보장하지 않는 자료형이다. add()는 원소 하나를 추가하고, remove()는 원소 하나를 삭제하며, update()는 여러 원소를 한 번에 추가한다. 함정: {'한국', '중국', '일본'}은 튜플이 아니라 set이다. asia.add('중국')은 이미 있는 값이라 변화가 없다. update({'한국', '홍콩', '태국'})에서 '한국'은 중복이라 추가되지 않고 '홍콩', '태국'만 추가된다. 예시: {'한국','중국','일본'} → 베트남 추가 → 중국 추가 무시 → 일본 삭제 → 한국/홍콩/태국 update →

[프로그래밍] 유형: Python map, lambda, list 변환 규칙: map(함수, 반복자료)는 반복자료의 각 원소에 함수를 하나씩 적용한다. lambda num: num + 100은 입력값 num에 100을 더해서 반환하는 익명 함수이다. 함정: map의 결과는 바로 리스트처럼 출력되지 않으므로 list()로 감싸서 리스트로 변환한다. 예시: [1,2,3,4,5]의 각 원소에 +100 적용 → [101,102,103,104,105]
