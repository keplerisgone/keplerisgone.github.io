정보처리기사 시나공 요약 ​

  • 애자일: 고객의 변화하는 요구사항, 고객과의 소통 중심, 사이클 중심, 절차, 문서, 계획 << 소통, 협업, 변화 대응
    • Scrum, XP, Kanban, crystal, FDD, ASD, DSDM
  • 리팩토링: 코드 품질 향상이 목적, 외부 기능의 변화 없이 내부 구조를 개선하는 작업 (이해하기 쉽게 만드는 것)
  • 요구사항 분류
    • 비기능적 요구사항: 성능, 보안, 품질, 안전 등에 대한 요구사항
    • 기능적 요구사항: 세스템의 기능, 제어 연산, 기술에 대한 요구사항
  • UML: 소프트웨어 공학에서 사용되는 표준화된 범용 모델링 언어
    • 사물, 관계, 다이어그램으로 구성
    • 사물: 구조(structural), 행동(behavioral), 그룹(Grouping), 주해(annotation)
      • 구조 사물은 다시 Class, Interface, Usecase, Node 등으로 나뉨
    • 관계: Assosiation(연관), Aggregation(집합), Composition(포함), Generalization(일반화), Dependency(의존), Realization(실체화) 관계로 나눌 수 있다.
    • 다이어그램:
      • 구조적 다이어그램: 클래스 다이어그램, 객체 다이어그램, 컴포넌트 다이어그램, 배치 다이어그램, 복합체 다이어그램, 패키지 다이어그램
      • 행위적 다이어그램: 유스케이스 다이어그램, 상태 다이어그램, 활동 다이어그램, 절차 다이어그램, 통신 다이어그램, 타이밍 다이어그램, 상호작용 개요 다이어그램
  • 비용 산정 기법:
    • 하향식 비용 산정 기법: 비과학적인 기법, 전문가 측정 기법, 델파이 측정 기법
    • 상향식 비용 산정 기법:
      • LOC: 낙관치, 기대치, 비관치 등을 기반으로 전체 비용을 산정하는 법
      • 예측치:
        • (낙관치+4x기대치+비관치)/6
      • line of code를 사용해 계산한다. - LOC/인당 월평균 생산 코드 라인/투입 인원
      • 단계별 노력 기법도 있다.
    • 수학적 산정 기법
      • COCOMO:
        • 보헴이 제안한 LOC 기반 비용 산정 기법
        • 규모에 따라 Basic, Intermediate, Detailed로 나뉨.
        • Basic은 KDSI에 따라 Organic - Semi-Detached - Embedded로 나뉨.
  • 데이터베이스 시스템
    • 데이터의 종류:
      • 공유 데이터(shered data): 공통으로 필요로 하는 데이터
      • 통합 데이터(integrated data): 불필요한 데이터를 제거하고 중복이 최소화된 데이터
      • 저장 데이터(stored data): 시스템이 접근할 수 있는 곳에 저장된 데이터
      • 운영 데이터(operational data): 조직의 목적을 위해 지속적으로 필요한 데이터
    • DB 시스템 특징: 실시간 접근성, 계속적인 진화, 동시 공유, 내용에 의한 참조, 데이터의 논리적 독립성
  • 스키마:
    • 데이터베이스의 자료 구조와 표현 방법, 관계 등을 정의한 제약조건이다.
    • 외부, 개념, 내부 스키마로 나뉜다.
    • 외부 스키마: 특정 사용자의 입장에서 필요한 데이터베이스 구조를 정의 (제일 바깥의 것)
    • 개념 스키마: 조직의 입장을 고려한 것
    • 내부 스키마: 물리적인 저장장치 입장을 고려한 것
  • 개념적 설계 -> 논리적 설계 -> 물리적 설계도 알아놓긴 해야 할 듯
  • 데이터 모델링의 구조:
    • 현실 세계의 개념적인 기업의 정보 구조를 디지털 세계의 논리적인 데이터 모델로 명확하고 체계적으로 변환해 문서화하는 기법을 말한다.
    • 표시 요소: (논리적) 구조, 연산, 제약 조건
    • 구성 요소: 개체, 속성, 관계
  • Cardinality: 튜플에서의 총 개체 수
  • Degree: 튜플에서의 총 속성 수
  • E-R 다이어그램 (ERD):
    • Peter chan이 최초로 제안, 개체, 관계, 속성으로 이루어짐.
    • 사각형이 개체, 원형은 속성, 마름모는 관계 타입
  • 키 (Key):
    • 데이터를 분류, 정렬, 검색할 때의 기준이 되는 속성을 의미
    • 후보키: 유일성과 최소성을 만족
    • 기본키: 후보키들 중 튜플의 식별을 위해 지정된 속성
    • 슈퍼키: 유일성을 만족, 최소성은 불만족
  • 관계 해석:
    • 관계 데이터의 연산을 표기하는 방법. 수학의 술어 해석에 기반한다.
    • 튜플 관계 해석, 도메인 관계 해석으로 나눌 수 있음.
  • 관계 대수

    • 선택(Selection):

      • σ: 조건을 만족하는 튜플, 즉 행을 선택하는 연산
      • σ학과='컴퓨터공학과'(학생)

    • 프로젝트(Projection):

      • π: 릴레이션에서 특정 속성, 즉 열만 추출하는 연산
      • π학번, 이름(학생)

    • 합집합(Union):

      • ∪: 두 릴레이션에 존재하는 모든 튜플을 합치는 연산
      • R ∪ S

    • 교집합(Intersection):

      • ∩: 두 릴레이션에 공통으로 존재하는 튜플만 추출하는 연산
      • R ∩ S

    • 차집합(Difference):

      • −: 첫 번째 릴레이션에는 존재하지만 두 번째 릴레이션에는 존재하지 않는 튜플을 추출하는 연산
      • R − S

    • 카티션 프로덕트(Cartesian Product):

      • ×: 두 릴레이션의 모든 튜플 조합을 만드는 연산
      • R × S

    • 조인(Join):

      • ⋈: 두 릴레이션에서 관련 있는 튜플을 조건에 따라 결합하는 연산
      • 학생 ⋈ 학생.학과번호 = 학과.학과번호 학과

    • 디비전(Division):

      • ÷: 두 릴레이션 A, B에 대해 B 릴레이션의 모든 조건을 만족하는 A의 튜플을 구하는 연산
      • 수강 ÷ 과목

    • 세타 조인(Theta Join):

      • ⋈θ: =, <, >, ≤, ≥, ≠ 등의 비교 연산자를 조건으로 사용하는 조인 연산
      • R ⋈R.A < S.B S

    • 동등 조인(Equi Join):

      • ⋈=: 조인 조건이 등호(=)인 조인 연산
      • 학생 ⋈학생.학과번호 = 학과.학과번호 학과

    • 자연 조인(Natural Join):

      • ⋈: 두 릴레이션에서 같은 이름의 속성을 기준으로 자동으로 조인하고 중복 속성은 제거하는 연산
      • 학생 ⋈ 학과

    • 왼쪽 외부 조인(Left Outer Join):

      • ⟕: 왼쪽 릴레이션의 모든 튜플을 보존하고, 오른쪽 릴레이션과 매칭되지 않는 값은 NULL로 채우는 연산
      • 학생 ⟕ 학과

    • 오른쪽 외부 조인(Right Outer Join):

      • ⟖: 오른쪽 릴레이션의 모든 튜플을 보존하고, 왼쪽 릴레이션과 매칭되지 않는 값은 NULL로 채우는 연산
      • 학생 ⟖ 학과

    • 완전 외부 조인(Full Outer Join):

      • ⟗: 양쪽 릴레이션의 모든 튜플을 보존하고, 매칭되지 않는 값은 NULL로 채우는 연산
      • 학생 ⟗ 학과
  • 정규화
    • 데이터 무결성을 유지하기 위해 중복성을 최소화하고 정보의 일관성을 보장하기 위한 개념
    • 이상 현상을 제거하기 위함:
      • 삽입 이상(관련 없는 정보도 함께 삽입)
      • 갱신 이상(데이터의 불일치가 발생하는 현상)
      • 삭제 이상(데이터를 삭제할 때 관련된 정보도 함께 삭제해야 하는 현상)
    • 도부이결다조
    • 반정규화: 일부러 중복, 분리 등을 수행하는 것
    • 비정규화: 정규화가 진행되지 않은 상태
  • 데이터 종속성:
    • 함수 종속: 한 속성이 다른 속성을 유일하게 식별할 수 있는 상태
      • 학번(결정자)은 중복되지 않기 때문에 학생의 다른 속성(종속자)들을 유일하게 식별한다.
    • 완전 함수 종속 (Full Functional Dependency): 종속자가 모든 결정자에 종속되는 경우
    • 부분 함수 종속 (Partial Functional Dependency): 종속자가 일부 결정자에 종속되는 경우
    • 이행적 함수 종속 (Transitive Dependency): 종속자 C의 결정자가 여러 개인 경우
  • 트랜잭션:
    • 데이터베이스를 조작하는 논리적 연산들이 하나 이상 모인 단위 작업이다.
    • 특징: 원자성, 일관성, 고립성, 지속성
  • 파일의 구조:
    • 순차, Index, 해싱
  • 오류 대응 방안:
    • 이중화: 복수의 시스템을 활용하여 주 시스템에 장애가 발생하는 경우에도 정상적인 서비스가 제공될 수 있도록 하기 위한 기술
      • 액티브-액티브, 액티브-스탠바이로 나뉨.
    • 재해 복구 시스템(DRS): Mirror site, Hot site, Warm site, Cold site
    • 복구 목표 시간(RTO)
    • 복무 목표 시점(RPO)
    • 업무 영향 분석(BIA)
  • 접근 통제 정책
    • DAC: 임의적 접근 통제, 신분에 따른 통제
    • MAC: 강제적 접근 통제, 전문가에 의해 생성되는 규칙에 기반을 두는 통제
    • RBAC: 역할 기반 접근 통제, 사용자의 역할에 기반을 둔 통제
  • 데이터 제어어(DCL)
    • 트랜잭션 복구 관련 용어

      • LOG

        • DBMS가 데이터베이스의 처리 내용, 이용 상황, 상태 변화 등을 시간 순서대로 기록한 것
        • 장애 발생 시 복구 작업의 근거가 된다.

      • REDO

        • 이미 완료된 트랜잭션의 작업을 다시 수행하는 연산
        • 트랜잭션의 start와 commit 기록이 모두 있는 경우 수행한다.
        • 로그를 이용하여 해당 데이터 항목을 이전 값에서 이후 값으로 변경한다.
        • 문제의 ①에 해당한다.

      • UNDO

        • 완료되지 않은 트랜잭션의 작업을 취소하는 연산
        • 트랜잭션의 start 기록은 있지만 commit 기록이 없는 경우 수행한다.
        • 로그를 이용하여 해당 데이터 항목을 이후 값에서 이전 값으로 되돌린다.
        • 문제의 ②에 해당한다.

      • COMMIT

        • 트랜잭션이 정상적으로 완료되었음을 확정하는 연산
        • COMMIT 이후의 결과는 데이터베이스에 영구 반영된다.

      • ROLLBACK

        • 트랜잭션 수행 중 오류가 발생했을 때 변경 내용을 취소하고 이전 상태로 되돌리는 연산
        • 트랜잭션의 원자성을 보장하기 위해 사용된다.

      • RECOVERY

        • 장애 발생 시 로그, 백업 등을 이용하여 데이터베이스를 정상 상태로 복구하는 작업
        • REDO와 UNDO가 대표적인 복구 연산이다.

      • BACKUP

        • 장애나 데이터 손실에 대비하여 데이터베이스의 사본을 별도로 저장하는 작업
        • 복구 작업의 기반 자료가 된다.

      • CHECK

        • 데이터베이스의 상태나 무결성 조건을 검사하는 작업
        • 트랜잭션 복구 문제에서는 핵심 정답보다는 보기용 용어로 나오는 경우가 많다.
  • 데이터 표현 방식: XML, JSON, YAML, AJAX

    • 라인 커버리지
    • 코드 커버리지: 구문 커버리지(모든 구문 테스트), 결정 커버리지(참과 거짓), 조건 커버리지(결정문 내부 조건식의 결과), 조건/결정 커버리지, 다중 조건 커버리지
  • 화이트박스 테스트 기법
    • 기초 경로(Basic Path) 테스트, 조건 검사, 루프 검사, 데이터 흐름 검사
  • 블랙박스 테스트 기법
    • 동등 분할(Equivalence Partitioning), 원인 효과-그래프(Cause-Effect Graphing), 오류 예측(Error Guessing), 비교(Comparison)
  • V-모델:
    • 요구사항 분석, 시스템 설계, 아키텍처, 모듈 설계, 구현, 단위 테스트, 통합 테스트, 시스템 테스트, 인수 테스트
  • 테스트 오라클:
    • 참 오라클, 샘플링 오라클, 휴리스틱 오라클, 일관성 검사 오라클
  • 애플리케이션 성능 측정:
    • 처리량, 응답 시간, 경과 시간, 자원 사용률
  • 데이터베이스 보안: 무결성, 가용성, 기밀성
  • 암호화 알고리즘:
    • 대칭키 알고리즘(키가 하나, 양방향, 비밀키): DES, SEED, AES, ARIA, IDEA, Skipjack, RC5, WEP, TKIP
    • 비대칭키 알고리즘(키가 두개, 단방향, 공개키): RSA, ECC, DSA
    • 해시 암호: MD 5, SHA
  • 보안 공격 유형

    • 웹 공격

      • SQL Injection: SQL 쿼리 삽입
      • XSS: 악성 스크립트 삽입
      • CSRF: 사용자의 인증 상태를 이용한 요청 위조
      • Directory Traversal: ../ 경로 조작
      • Command Injection: OS 명령어 삽입

    • 네트워크 공격

      • Sniffing: 패킷 도청
      • Spoofing: 주소/신분 위조
      • ARP Spoofing: IP-MAC 매핑 위조
      • DNS Spoofing: DNS 응답 위조
      • Session Hijacking: 세션 탈취

    • 서비스 거부 공격

      • DoS: 단일 출처 서비스 마비
      • DDoS: 분산 출처 서비스 마비
      • SYN Flooding: SYN 대량 전송
      • Smurf: ICMP + 브로드캐스트 + IP 위조
      • RUDY: 느린 HTTP POST
      • Slowloris: 느린 HTTP Header

    • 악성코드

      • Virus: 파일 감염
      • Worm: 자기 복제, 네트워크 전파
      • Trojan Horse: 정상 프로그램 위장
      • Ransomware: 파일 암호화 후 금전 요구
      • Spyware: 사용자 정보 수집
      • Rootkit: 관리자 권한 은닉
      • Key Logger: 키 입력 기록

    • 보안 프로토콜

      • SSL/TLS: 전송 계층 보안, HTTPS 기반
      • HTTPS: HTTP + SSL/TLS
      • S-HTTP: HTTP 메시지 단위 보안
      • IPsec: IP 계층 보안, VPN
      • SSH: 안전한 원격 접속
      • SFTP: SSH 기반 파일 전송
      • PGP/S-MIME: 이메일 암호화/전자서명
      • Kerberos: 티켓 기반 인증
      • RADIUS/TACACS+: AAA 인증
      • WPA/WPA2/TKIP: 무선 LAN 보안
      • DNSSEC: DNS 위조 방지
  • 프로세스 상태:
  • 프로세스 상태 전이
  • 스케줄링(SJF, HRN, SRN 등등)
  • IPv4, IPv6 (서브네팅)
  • OSI 7계층
  • 프로토콜의 기본 요소 3가지 및 프로토콜의 의미
  • RAID
