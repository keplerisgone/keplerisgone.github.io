SQL ​

INSERT, SELECT, UPDATE 기본 문법 ​

  • 유형: INSERT, SELECT, UPDATE문의 기본 문법에 들어갈 키워드를 찾는 문제
  • 규칙: 직접 값을 추가할 때 VALUES, 조회 결과를 추가할 때 SELECT, 조회 대상은 FROM, 수정값 지정은 SET을 사용한다.
  • 함정: INSERT INTO 뒤에는 VALUES뿐 아니라 SELECT문도 올 수 있으며, UPDATE문은 변경할 열 앞에 SET을 사용한다.
  • 예시: INSERT INTO 테이블(열) VALUES(값), INSERT INTO 테이블(열) SELECT 열 FROM 테이블 형태로 사용한다.
  • 메모: INSERT VALUES, SELECT FROM, UPDATE SET으로 묶어 암기한다.

네트워크 ​

IPSec ​

  • 유형: 네트워크 계층에서 사용하는 보안 프로토콜의 특징을 보고 명칭을 찾는 문제
  • 규칙: IPSec은 IP 패킷을 인증·암호화하며 AH와 ESP를 사용해 VPN을 구현한다.
  • 함정: SSL/TLS는 주로 전송 계층 위에서 동작하지만 IPSec은 네트워크 계층의 IP 패킷 자체를 보호한다.
  • 예시: 기업의 원격 접속 VPN이나 지점 간 VPN에서 IPSec을 사용할 수 있다.
  • 메모: IP + Security, AH·ESP·VPN이 나오면 IPSec.

패킷 교환 방식 ​

  • 유형: 패킷 교환 방식에서 연결형과 비연결형을 구분하는 문제
  • 규칙: 연결형은 통신 전에 경로를 설정하는 가상 회선 방식이고, 비연결형은 패킷마다 독립적으로 전송하는 데이터그램 방식이다.
  • 함정: 회선 교환이 아니라 패킷 교환 내부의 두 방식인 가상 회선과 데이터그램을 묻는 문제이다.
  • 예시: 가상 회선은 모든 패킷이 같은 경로를 따르고, 데이터그램은 패킷마다 서로 다른 경로를 사용할 수 있다.
  • 메모: 연결형 = 가상 회선, 비연결형 = 데이터그램.

RIP 홉 카운트 ​

  • 유형: RIP의 홉 카운트를 기준으로 최단 경로를 선택하는 문제
  • 규칙: RIP은 각 링크의 가중치 합이 아니라 목적지까지 거치는 홉 수가 가장 적은 경로를 선택한다.
  • 함정: 그림의 간선 숫자를 더하면 A -> D -> C -> E -> F가 작지만, RIP 기준에서는 3홉인 A -> D -> C -> F가 선택된다.
  • 예시: A -> D -> C -> F는 3홉이고 A -> D -> C -> E -> F는 4홉이므로 전자가 최단 경로이다.
  • 메모: 가중치 최단 경로와 RIP의 홉 카운트를 혼동하지 않는다.

테스트와 보안 ​

AES ​

  • 유형: 키 길이와 블록 크기 특징을 보고 대칭키 암호 알고리즘을 찾는 문제
  • 규칙: AES는 128비트 블록을 사용하며 키 길이는 128·192·256비트 중 하나이다.
  • 함정: 1997년은 NIST가 AES 공모를 시작한 시점이며, 최종 표준은 Rijndael을 기반으로 2001년에 제정되었다.
  • 예시: DES를 대체하는 현대적인 대칭키 블록 암호로 파일·통신 데이터 암호화에 널리 사용된다.
  • 메모: AES는 128비트 고정 블록, 키는 128·192·256비트.

통합구현과 신기술 ​

순차적 응집도 ​

  • 유형: 모듈 내부 기능들의 관계를 보고 응집도 종류를 구분하는 문제
  • 규칙: 한 기능의 출력이 다음 기능의 입력으로 사용되면 순차적 응집도이다.
  • 함정: 실행 순서가 관련된다는 표현만 보면 절차적 응집도처럼 보이지만, 출력→입력 관계가 있으면 순차적 응집도이다.
  • 예시: 데이터를 읽기 → 가공하기 → 가공된 데이터를 출력하기
  • 메모: 응집도는 우연적 → 논리적 → 시간적 → 절차적 → 통신적 → 순차적 → 기능적 순으로 강해진다.

Iterator 패턴 ​

  • 유형: 디자인 패턴의 특징을 보고 해당 행위 패턴을 찾는 문제
  • 규칙: Iterator는 컬렉션의 내부 구조를 노출하지 않고 요소를 순차적으로 접근하게 한다.
  • 함정: 컬렉션을 다룬다고 Composite를 고르는 것이 아니라 반복·순차 접근이 핵심이면 Iterator이다.
  • 예시: 리스트의 구현 방식을 몰라도 hasNext()와 next()로 요소를 차례대로 조회한다.
  • 메모: 내부 구조 숨김 + 순차 접근 = Iterator.

운영체제 ​

SRT와 비선점형 SJF ​

  • 유형: 도착 시간이 다른 프로세스에서 SRT와 비선점형 SJF의 실행 순서를 비교하는 문제
  • 규칙: SRT는 새 프로세스가 도착할 때 현재 프로세스의 남은 시간보다 짧으면 즉시 선점한다.
  • 함정: SJF는 실행을 시작한 프로세스를 끝까지 수행하지만 SRT는 도착한 프로세스에 의해 실행 순서가 바뀔 수 있다.
  • 예시: 시간 1에 B가 도착하면 B의 실행시간 4가 A의 남은 시간 7보다 짧으므로 A를 선점한다.
  • 메모: 모든 프로세스가 동시에 도착하거나 더 짧은 프로세스가 중간에 도착하지 않을 때만 SRT와 SJF 결과가 같을 수 있다.

Java 배열 참조 비교 ​

java
class Main {
    public static void main(String[] args) {
        int[] a = new int[]{1, 2, 3, 4};
        int[] b = new int[]{1, 2, 3, 4};
        int[] c = new int[]{1, 2, 3};

        check(a, b);
        check(a, c);
        check(b, c);
    }

    public static void check(int[] a, int[] b) {
        if (a == b) {
            System.out.print("O");
        } else {
            System.out.print("N");
        }
    }
}
  • 유형: Java 배열에서 == 연산자가 비교하는 대상을 묻는 문제
  • 규칙: 배열에 ==를 사용하면 내부 원소가 아니라 두 변수가 같은 배열 객체를 가리키는지 비교한다.
  • 함정: a와 b의 원소는 같지만 각각 new int[]로 생성된 별개의 객체이므로 a == b는 false이다.
  • 예시: int[] b = a;로 작성했다면 a == b가 true가 되어 첫 출력이 O가 된다.
  • 메모: Java 배열의 ==는 참조값 비교이므로 세 비교 모두 N이다.
