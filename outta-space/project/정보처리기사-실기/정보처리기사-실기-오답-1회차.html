📝 정처기 실기 오답노트 요약 정리 (취약 개념 중심) ​

1. 테스트 (Testing) ​

  • 화이트박스 테스트 vs 블랙박스 테스트
    • 화이트박스 테스트: 소스코드의 내부 논리적인 구조를 커버하도록 테스트 케이스를 설계 (기초 경로 검사, 제어 구조 검사, 조건 검사, 루프 검사, 데이터 흐름 검사).
    • 블랙박스 테스트: 내부 구조를 보지 않고 요구사항 명세서를 기반으로 기능 중심 테스트를 수행 (동치 분할 검사, 경계값 분석, 원인-효과 그래프, 오류 예측, 비교 검사).
  • 테스트 오라클의 종류와 특징: 테스트 결과를 판단하기 위한 기법으로 샘플링 오라클, 일관성 검사 오라클, 참 오라클이 존재하며, 특징으로는 제한된 검증, 수학적 기법, 자동화 기능이 있음.
  • 일반적인 테스트 프로세스 순서: 계획 → 분석 및 디자인 → 케이스 및 시나리오 작성 → 수행 → 결과 평가 및 보고서 작성 → 결함 추적 및 관리.
  • 단위 테스트 수행 방법: 명세 기반 테스트(블랙박스)와 구조 기반 테스트(화이트박스)로 나뉨.
  • 하향식(Top Down) 통합 테스트: 상위 모듈에서 하위 모듈 방향으로 통합하며, 깊이 우선/넓이 우선 통합법을 사용해 초기부터 시스템 구조를 시각화할 수 있음.
  • 워크스루 (Walk Through): 프로그램을 실행하지 않고 검토 회의 전에 미리 준비된 자료를 배포하여 사전 검토를 진행하는 정적 테스트.

2. SQL 문법 ​

  • 그룹 함수 및 조건: AVG() 함수를 사용할 때는 GROUP BY와 함께 사용해야 함.
  • NULL 체크: IS NOT NULL, IS NULL 구문을 사용.
  • 패턴 매칭: 특정 문자열이 포함되는 조건은 LIKE '%길' 형태로 사용.
  • 범위 조건: A와 B 사이의 값은 BETWEEN A AND B로 지정.
  • 정렬: 내림차순 출력은 ORDER BY 컬럼명 DESC (오름차순은 ASC).
  • DCL (권한 부여): GRANT SELECT ON 테이블명 TO 사용자명 WITH GRANT OPTION; (WITH GRANT OPTION은 권한 재부여 가능 옵션).
  • DDL (인덱스 생성): CREATE UNIQUE INDEX 인덱스명 ON 테이블(컬럼 ASC);.
  • DML (데이터 삽입): INSERT INTO 테이블(컬럼1, 컬럼2) VALUES (값1, 값2);.
  • 뷰 생성: CREATE VIEW 뷰명(컬럼명들) AS SELECT ...;.
  • 연산자 용법: 여러 값 중 하나에 해당하는 조건은 IN (1, 2, 3)을 사용하며, 같지 않음은 <> 로 표현함.

3. 데이터베이스 (Database) ​

  • 정규화 (Normalization) 암기 공식: "도부이결다조"
    • 제1정규형: 도메인이 원자값
    • 제2정규형: 부분 함수 종속 제거
    • 제3정규형: 이행적 함수 종속 제거
    • BCNF: 결정자이면서 후보키가 아닌 함수 종속 제거 (보이스-코드 정규형)
    • 제4정규형: 다치 종속 제거
    • 제5정규형: 조인 종속 제거
  • SQL 언어의 분류
    • DDL (데이터 정의어): CREATE, ALTER, DROP.
    • DML (데이터 조작어): SELECT, INSERT, DELETE, UPDATE.
    • DCL (데이터 제어어): 보안, 무결성, 트랜잭션 제어를 담당하며 COMMIT, ROLLBACK, GRANT, REVOKE가 있음.
  • 삭제 옵션 (RESTRICT): 테이블 삭제 시 해당 테이블을 참조하는 테이블이 있다면 삭제를 취소하는 옵션.
  • 파티셔닝 분할 유형: 대용량 테이블을 나누는 기준으로 범위(Range), 해시(Hash), 조합(Composite), 목록(List) 분할이 있음.
  • 관계대수 (디비전, ÷): B 릴레이션의 모든 조건을 만족하는 튜플을 A에서 분리해 프로젝션하는 연산 기호.
  • 반정규화: 성능 향상과 운영 단순화를 위해 정규화된 엔터티, 속성 등을 중복, 통합, 분리하는 기법.

4. 보안 (Security) ​

  • 정보보호의 5대 목표 (무기가인책): 기밀성, 무결성, 가용성, 인증성, 책임성.
  • 컴퓨터 바이러스 공격 유형: 가로막기, 가로채기, 수정, 위조.
  • 접근 통제 정책
    • MAC (강제적): 시스템이 보안 등급을 기준으로 부여.
    • DAC (임의적): 데이터 소유자가 신분을 기준으로 부여.
    • RBAC (역할 기반): 중앙 관리자가 역할을 기준으로 부여.
  • 암호화 및 보안 기술
    • Hash (해시): 임의의 길이 입력을 고정된 길이의 값이나 키로 변환.
    • 블록 암호: 평문을 일정한 블록 단위로 나누어 암호화.
    • 스트림 암호: 비트/바이트 단위로 난수 초기값을 발생시켜 XOR 연산 수행 (빠르고 오류 영향 적음).
    • 시큐어 코딩: 초기 설계 단계부터 보안 취약점을 사전에 제거하도록 코딩하는 방식.
  • 버퍼 오버플로우 방지 기법: 복귀 주소와 변수 사이에 특정 값을 저장하는 스택 가드, 함수 종료 시 복사본과 비교하는 스택 실드.
  • 바이러스/악성코드 개념
    • 그레이웨어 (Grayware): 유용하지 않거나 악의적일 수 있는 애드웨어, 스파이웨어 등 악성 소프트웨어의 총칭.
    • 바이러스 감염 위치별 분류: 부트, 파일, 매크로, 메모리 상주.
    • IPS (침입 차단 시스템): 내부 네트워크 보호를 위해 외부 침입을 통제하는 시스템.

5. 프로그래밍 (Programming) ​

  • C언어 연산자 우선순위: 덧셈(+) 연산자가 동등 비교(==) 연산자보다 우선순위가 높음.
  • 객체지향 및 Java 개념
    • 상속과 메서드 호출: 상위 클래스의 인스턴스로는 하위 클래스에만 정의된 메서드를 호출할 수 없음.
    • Encapsulation (캡슐화): 내부 변수 접근을 private로 막고 메서드를 통해 접근하게 하는 기법.
    • this 와 super: 내 객체의 필드 접근은 this.필드 = 매개변수, 부모 클래스 생성자 호출은 super(인수) 사용.
    • 오버라이딩: 부모에게 상속받은 메서드를 자식 클래스에서 재정의하는 기술.
  • Python 특징
    • self: 클래스 메서드 내부에서 객체 자신을 지칭할 때 사용.
    • 들여쓰기: 중괄호 {} 대신 코드의 지역(블록)을 구분할 때 사용.

6. 운영체제 (OS) ​

  • 운영체제 스케줄링 및 상태
    • 프로세스 상태 전이: 준비→실행(Dispatch), 실행→준비(Time Run Out), 실행→대기(Block), 대기→준비(Wake Up).
    • 비선점형 스케줄링: 강제 중단이 불가능한 방식으로 FIFO, SJF, HRN이 있음.
  • 교착상태 (Deadlock)
    • 발생 조건: 상호 배제, 점유와 대기, 비선점, 환형 대기.
    • 회복 시 희생양 기준: 우선순위가 낮거나, 진행률이 적거나, 자원을 적게 쓰거나, 무한 대기 등으로 수행 불가능한 프로세스.
  • 메모리 및 자원 관리
    • 오버레이 기법: 주기억장치보다 큰 프로그램을 실행하기 위해 프로그램을 조각내어 프로그래머가 직접 메모리에 적재하는 기법.
    • 스와핑: 보조기억장치에서 주기억장치로 불러오는 Swap In, 반대로 내보내는 Swap Out.
    • 레지스터: **제어장치(CU)**는 명령어 수행 데이터 저장, **연산장치(ALU)**는 산술/논리 연산 수행.
  • 유닉스(Unix): C언어 기반, 높은 이식성, 다중 사용자 지원. 핵심 요소로 커널, 쉘, 유틸리티가 있음.
  • 커널의 종류: 최소 기능만 담은 마이크로 커널, 동적 적재가 불가하지만 빠르고 구현이 간단한 모놀리식 커널.
  • 쉘 스크립트: 여러 제어 명령을 자동으로 실행할 수 있는 인터프리터 방식의 프로그램 (.sh).

7. 네트워크 (Network) ​

  • OSI 7계층 암기
    • 상위 3계층: 응용, 표현, 세션 계층.
    • 하위 4계층: 전송, 네트워크, 데이터링크, 물리 계층.
    • 데이터링크 계층: MAC 주소 사용, Frame 단위 전송, 신뢰성 있는 노드 간 전송 및 오류/흐름 제어.
    • 물리 계층 장비: 통신 케이블, 허브, 리피터.
  • IP 및 주소 체계
    • 서브넷 마스크: 고정된 크기 분할은 FLSM, 다양한 크기 분할은 VLSM.
    • 도메인 네임 (Domain Name): 숫자 IP를 외우기 쉽게 문자 형태로 변환한 주소.
    • CIDR / DHCP: 클래스를 무시하고 유연하게 IP를 할당하거나 동적으로 할당하는 프로토콜.
    • ARP / RARP: IP를 MAC 주소로 변환하는 것은 ARP, 그 반대(MAC을 IP로)는 RARP.
  • 네트워크 통신망 및 프로토콜
    • 버스 종류: 외부 버스는 주소 버스, 데이터 버스, 제어 버스로 나뉨.
    • TCP 제어 비트 (플래그): 연결을 처음 개설할 때 사용하는 제어 비트는 SYN.
    • 전자우편 (Email): SMTP, POP3, MIME, IMAP.
    • FTP: 업로드/다운로드를 지원하며 ASCII와 Binary 전송 모드를 가짐.
    • SNMP: 장비 관리 감시를 위한 UDP 기반 응용 계층 프로토콜.
    • RTP: 실시간 오디오/비디오(스트리밍) 전달 프로토콜.
  • 변조 방식: 1과 0을 위상 차이로 전송하는 방식은 위상 편이 변조 (PSK).

8. IT 신기술 및 소프트웨어 공학 용어 ​

  • 개념 및 방법론
    • 클린 코드 원칙: 중복 최소화를 위해 가독성, 단순성, 의존성 배제, 중복성 제거, 추상화 원칙을 따름.
    • 분산 시스템 투명성: 위치, 중복, 병행, 장애 투명성.
  • IT 신기술 용어
    • SOA (서비스 지향 아키텍처): 소프트웨어 인프라를 재사용 가능한 서비스 단위로 구축.
    • DRM (디지털 저작권 관리): 디지털 콘텐츠의 안전한 배포 및 저작권 보호 시스템.
    • TrustZone: 프로세서 내 일반 구역과 보안 구역을 물리적으로 분리하는 기술.
    • N-Screen: 여러 단말기에서 동일 콘텐츠를 이어서 볼 수 있는 서비스.
    • Thin Client PC: 하드디스크 없이 메모리만 갖추고 서버 네트워크로 동작하는 PC.
    • 데이터 기술: 데이터를 분석하기 좋게 축적하는 Data Warehouse, 방대한 상호작용 데이터인 Broad Data, 데이터를 설명하는 Meta Data, 불필요한 데이터를 압축하여 분리 보관하는 Data Diet, 웹상 무료 공개 데이터인 Linked Open Data 등.
    • 디지털 아카이빙: 아날로그/디지털 자원을 변환, 압축, 메타데이터화 하여 장기 보존.
    • Hadoop / Tajo: 대규모 분산 데이터 처리를 위한 자바 프레임워크(Hadoop)와 국내 주도의 SQL 기반 분산 DW 프로젝트(Tajo).
    • 하드웨어 및 네트워크 기술: 전원 없이 상태를 기억하는 멤리스터 (Memristor), 위아래 구분 없는 C-Type USB, 조각 방식으로 영구 보관하는 M-DISC, 대규모 무선 디바이스 최적화 기술 Mesh Network, 스마트폰 근거리 무선 인식 비콘 (Beacon), 블루투스/UWB 기반 소규모 무선 통신망 PICONET, 발행-구독 기반 경량 IoT 통신 프로토콜 MQTT.

9. (추가) 도부이결다조 ​

  • 도 (제1정규형, 1NF) : 도메인이 원자값 릴레이션에 속한 모든 속성과 도메인이 더 이상 분해할 수 없는 원자값만 가지도록 테이블을 분해한 상태를 의미합니다.
  • 부 (제2정규형, 2NF) : 부분 함수 종속 제거 제1정규형을 만족하는 상태에서 부분 함수 종속을 제거한 릴레이션 스키마입니다. 기본키가 아닌 모든 속성들이 기본키에 완전 함수 종속 관계를 갖도록 기본키를 중심으로 릴레이션을 분해합니다.
  • 이 (제3정규형, 3NF) : 이행적 함수 종속 제거 제2정규형을 만족하면서 이행적 함수 종속(Transitive Dependency)을 제거한 상태입니다. 결정자이자 종속자에 해당하는 속성을 기준으로 릴레이션을 분할합니다.
  • 결 (BCNF, 보이스-코드 정규형) : 결정자이면서 후보키가 아닌 종속 제거 제3정규형을 만족하면서, 결정자가 후보키가 아닌 함수 종속을 제거한 상태입니다. 이 단계에서는 릴레이션에 존재하는 모든 결정자가 후보키인 조건을 만족해야 합니다.
  • 다 (제4정규형, 4NF) : 다치 종속 제거 BCNF를 만족하면서, 둘 이상의 속성을 가진 결정자에 의해 정해지는 다치 종속(Multi-Valued Dependency)을 제거한 릴레이션 스키마입니다.
  • 조 (제5정규형, 5NF) : 조인 종속 제거 제4정규형을 만족하면서, 후보키를 통하지 않은 조인 종속(Join Dependency)을 제거한 상태입니다. 릴레이션에 존재하는 모든 조인 종속성이 릴레이션의 후보키를 통해서만 성립되도록 분해 전 릴레이션으로 재구성될 수 있게 만듭니다.
