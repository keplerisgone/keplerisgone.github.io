네트워크 ​

오류 제어와 오류 검출 ​

  • 유형: 오류 제어 방식인 FEC·BEC와 오류 검출 기법인 Parity·CRC를 구분하는 문제
  • 규칙: Hamming은 오류 검출·수정, FEC는 수신 측 자체 수정, BEC는 송신 측 재전송 요구 방식이다.
  • 함정: 재전송을 요구하는 동작은 ARQ지만 보기에는 ARQ가 없으므로 상위 오류 제어 방식인 BEC를 선택한다.
  • 예시: Parity는 1비트 검사 비트를 추가하고, CRC는 송수신 측이 동일한 생성 다항식을 사용한다.
  • 메모: Hamming → FEC, Parity·CRC → BEC의 소속 관계를 묶어서 암기한다.

데이터베이스 ​

관계대수 나눗셈 ​

  • 유형: 관계대수의 나눗셈 연산 R % S 결과를 구하는 문제
  • 규칙: R % S는 S의 모든 속성값과 대응되는 R의 나머지 속성값만 반환한다.
  • 함정: S의 값 중 하나만 만족하는 것이 아니라 b1과 b3을 모두 가진 A만 선택해야 한다.
  • 예시: a1은 (a1, b1), (a1, b3)이 모두 있으므로 결과에 포함되고 a2는 제외된다.
  • 메모: %는 관계대수의 나눗셈 연산으로, “S의 모든 조건을 만족하는 값”을 찾는 연산이다.

테스트와 보안 ​

OAuth ​

  • 유형: 제3자 서비스에 접근 권한을 위임하는 인증·인가 기술의 명칭
  • 규칙: OAuth는 비밀번호를 전달하지 않고 Access Token을 이용해 사용자가 허용한 범위의 권한만 제3자에게 위임한다.
  • 함정: OAuth 자체는 인증이 아니라 인가 프로토콜이며, 로그인 기능은 보통 OpenID Connect와 함께 구현된다.
  • 예시: 구글 계정으로 외부 서비스에 로그인하고 프로필 정보 접근을 허용하는 방식
  • 메모: 비밀번호 대신 토큰으로 권한을 위임하면 OAuth다.

SQL ​

CROSS JOIN과 LIKE ​

  • 유형: CROSS JOIN으로 모든 행 조합을 만든 뒤 LIKE 조건을 만족하는 행의 개수를 구하는 문제
  • 규칙: CROSS JOIN 결과 행 수는 두 테이블의 행 수를 곱하며, 이후 WHERE 조건으로 필요한 조합만 필터링한다.
  • 함정: CROSS JOIN 결과가 3×2=6개라고 바로 답하지 않고, A.NAME LIKE B.RULE 조건을 적용해야 한다.
  • 예시: smith는 s%와 %t%를 모두 만족하고, allen은 모두 불만족하며, scott도 두 조건을 모두 만족한다.
  • 메모: CROSS JOIN은 조인 조건 없이 가능한 모든 행 조합을 만드는 카티션 곱이다.

C 구조체 포인터와 문자열 포인터 연산 ​

c
#include <stdio.h>

struct Test {
    int i;
    const char *g;
};

int main() {
    struct Test test[] = {
        {1, "AB"},
        {2, "DC"},
        {3, "EB"}
    };

    struct Test *p = &test[1];

    printf("%s", p->g + (p->i - 1));

    return 0;
}
  • 유형: C 구조체 포인터와 문자열 포인터 연산의 출력 결과 추적
  • 규칙: char* 포인터에 정수를 더하면 문자의 값이 아니라 문자열에서 가리키는 위치가 이동한다.
  • 함정: p->g + 1은 문자 'D'에 1을 더하는 것이 아니라 "DC"의 두 번째 문자 'C'를 가리키는 포인터가 된다.
  • 예시: "DC" + 1은 'C'부터 시작하는 문자열 "C"를 의미한다.
  • 메모: 문자값 연산은 *(p->g) + 1처럼 역참조해야 한다.

C 연결 리스트, 누적 연산, XOR ​

c
#include <stdio.h>
 
struct Node {
    struct Node* next;
    unsigned int x;
};
 
int main() {
    struct Node t1 = { 0, 5u };
    struct Node t2 = { 0, 7u };
    struct Node t3 = { 0, 11u };
 
    t3.next = &t2;
    t2.next = &t1;
 
    struct Node* curr = &t3;
    int sum = 0;
 
    while (curr) {
        sum = sum * 3 + curr->x;
        curr = curr->next;
    }
 
    sum = (sum ^ 42u) + 100u;
 
    printf("%u\n", sum);
}
  • 유형: 연결 리스트 순회, 누적 연산, 비트 XOR, 정수 리터럴 접미사
  • 규칙: 숫자 뒤의 u는 unsigned int 상수이며, curr->x를 순서대로 읽어 sum = sum * 3 + x를 계산한다.
  • 함정: ^는 거듭제곱이 아니라 비트 XOR 연산자이며, 5u, 42u, 100u의 u는 값이 아니라 자료형만 unsigned로 지정한다.
  • 예시: 누적 결과 125에 대해 125 ^ 42 = 87, 87 + 100 = 187이다.
  • 메모: u는 1.0f의 f처럼 리터럴의 자료형 접미사다.

Java 인터페이스 구현 ​

  • 유형: Java에서 클래스가 인터페이스를 구현할 때 사용하는 키워드
  • 규칙: 클래스가 인터페이스를 구현할 때는 implements를 사용하고, 클래스를 상속할 때는 extends를 사용한다.
  • 함정: interface Machine을 상속한다고 생각해 extends를 쓰기 쉽지만, 클래스가 인터페이스를 따르는 경우는 implements다.
  • 예시: class WashingMachine implements Machine { ... }
  • 메모: 클래스→클래스는 extends, 클래스→인터페이스는 implements로 구분한다.

Python enumerate ​

  • 유형: Python의 enumerate, 리스트 순회, 딕셔너리·튜플 생성
  • 규칙: enumerate(data)는 각 요소를 (인덱스, 값) 형태로 반환하며 기본 인덱스는 0부터 시작한다.
  • 함정: index는 리스트 내부의 값이 아니라 바깥쪽 리스트의 위치이며, (list_sum, list_len)은 튜플이다.
  • 예시: 첫 반복에서 index = 0, lis = [3, 5, 2, 4, 1]이므로 result[0] = (15, 5)가 저장된다.
  • 메모: enumerate는 인덱스와 요소를 동시에 꺼내는 함수다.

C 연산자 우선순위 ​

  • 유형: C의 나머지·삼항·비트 시프트·비트 AND·논리 AND 연산 순서 추적
  • 규칙: 시프트 연산자는 비트 AND보다 먼저 계산되고, 삼항 연산자는 조건식 결과에 따라 한쪽 식만 실행한다.
  • 함정: z & z >> 1은 (z & z) >> 1이 아니라 z & (z >> 1)로 계산한다.
  • 예시: 2는 이진수 10이고 2 >> 1은 01이므로 10 & 01은 00이다.
  • 메모: 각 대입문이 끝날 때마다 z의 값을 갱신해 추적한다.

Java enum ​

java
enum Tri {
    A("A"), B("AB"), C("ABC");

    private String code;

    Tri(String code) {
        this.code = code;
    }

    public String code() {
        return code;
    }
}

public class Main {
    public static void main(String[] args) {
        Tri t = Tri.values()[Tri.A.name().length()];
        System.out.print(t.code());
    }
}
  • 유형: Java enum의 values(), name(), 배열 인덱스, 사용자 정의 메서드 실행 흐름
  • 규칙: values()는 enum 상수를 선언 순서대로 배열로 반환하고, name()은 상수 이름을 문자열로 반환한다.
  • 함정: Tri.A.name().length()의 결과는 1이며, 배열 인덱스 1은 두 번째 상수 B를 의미한다.
  • 예시: Tri.values()[1]은 Tri.B이고, Tri.B.code()는 "AB"를 반환한다.
  • 메모: values(), name(), code()가 각각 무엇을 반환하는지 구분한다.
