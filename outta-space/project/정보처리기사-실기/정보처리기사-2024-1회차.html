정보처리기사 2024년 1회차 요약 ​

네트워크 ​

서브넷 마스크와 호스트 주소 ​

  • 유형: 서브넷 마스크를 이용해 같은 네트워크의 할당 가능한 호스트 주소를 고르는 문제
  • 규칙: 같은 네트워크에 속하면서 네트워크 주소와 브로드캐스트 주소가 아닌 IP를 선택한다.
  • 함정: 마지막 옥텟이 .0이라고 항상 네트워크 주소인 것은 아니다. 전체 호스트 비트가 모두 0인지 확인해야 한다.
  • 예시: 129.200.10.16/22의 네트워크 범위는 129.200.8.0 ~ 129.200.11.255이고, 할당 가능 범위는 129.200.8.1 ~ 129.200.11.254이다.
  • 메모: 보기의 .0 주소들은 모두 네트워크 주소를 고르게 유도하는 함정으로 본다.

OSPF ​

  • 유형: 링크 상태 라우팅 프로토콜의 특징을 보고 명칭을 찾는 문제
  • 규칙: OSPF는 자율 시스템 내부에서 사용하는 링크 상태 기반 IGP이며, Dijkstra 알고리즘으로 최단 경로를 계산한다.
  • 함정: Dijkstra만 보고 판단하지 말고 링크 상태, 내부 게이트웨이 프로토콜, 단일 AS 조건을 함께 확인한다.
  • 예시: 각 라우터가 링크 상태 정보를 공유한 뒤 최단 경로 트리를 계산한다.
  • 메모: OSPF = 링크 상태 + IGP + Dijkstra.

데이터베이스 ​

정규형 판별 ​

  • 유형: 복합키와 함수적 종속 관계를 보고 가장 높은 정규형을 판별하는 문제
  • 규칙: 3NF는 부분 함수 종속과 비키 속성 간 이행 함수 종속이 제거된 상태이다.
  • 함정: 강사번호가 강좌명을 결정하지만 강사번호는 행 전체를 결정하는 슈퍼키가 아니므로 BCNF는 만족하지 않는다.
  • 예시: P001 -> 영어회화, P004 -> 기초토익이면 강사번호 -> 강좌명이 성립한다.
  • 메모: 1NF 원자값, 2NF 부분 종속 제거, 3NF 이행 종속 제거, BCNF 모든 결정자가 후보키.

세타 조인, 동등 조인, 자연 조인 ​

  • 유형: 조인 조건과 중복 속성 처리 방식에 따라 조인의 종류를 구분하는 문제
  • 규칙: 세타 조인은 비교 연산자를 사용하고, 동등 조인은 =만 사용하며, 자연 조인은 동등 조인 결과에서 중복 속성을 제거한다.
  • 함정: 동등 조인과 자연 조인은 조건은 같지만 자연 조인만 동일한 조인 속성을 한 번만 표시한다.
  • 예시: R.A = S.A로 결합하면 동등 조인이고, 결과에서 중복된 A 열을 제거하면 자연 조인이다.
  • 메모: 세타 조인 ⊃ 동등 조인 → 중복 속성 제거 시 자연 조인.

운영체제 ​

LRU와 LFU 페이지 교체 ​

  • 유형: 페이지 프레임 3개에서 LRU와 LFU의 페이지 부재 횟수를 계산하는 문제
  • 규칙: LRU는 가장 오래 참조되지 않은 페이지를, LFU는 참조 횟수가 가장 적은 페이지를 교체한다.
  • 함정: 두 알고리즘의 기준은 다르지만 이 참조 순서에서는 교체 대상이 우연히 같아 결과도 동일하다.
  • 예시: 4가 들어올 때 LRU는 오래된 3을, LFU는 참조 횟수가 1회인 3을 제거한다.
  • 메모: LRU와 LFU가 같은 알고리즘은 아니다. 이 문제에서만 페이지 부재 횟수가 모두 6회로 같다.

테스트와 보안 ​

MC/DC ​

  • 유형: 여러 조건이 포함된 결정문에서 각 조건의 독립적 영향 여부를 검사하는 커버리지 문제
  • 규칙: MC/DC는 각 개별 조건이 다른 조건과 독립적으로 전체 결정 결과를 바꾸는 경우를 최소 한 번 검증한다.
  • 함정: 세 문장에 각각 다른 답을 쓰는 문제가 아니라 모두 하나의 변경 조건/결정 커버리지를 설명한다.
  • 예시: A && B에서 B를 고정하고 A만 변경했을 때 전체 결과가 바뀌는 테스트 쌍을 만든다.
  • 메모: MC/DC는 모든 조합이 아니라 각 조건의 독립적인 영향만 확인한다.

루트킷 ​

  • 유형: 악성코드의 동작 특징을 보고 종류를 구분하는 문제
  • 규칙: 루트킷은 공격자의 존재와 악성 행위를 숨기면서 시스템의 관리자 권한 접근을 유지하는 도구이다.
  • 함정: 몰래 정보를 수집하는 Spyware가 아니라 시스템 콜 조작과 백신 탐지 우회가 핵심인 Rootkit이다.
  • 예시: 파일, 프로세스, 네트워크 연결을 숨기거나 운영체제의 시스템 콜을 변조한다.
  • 메모: Rootkit은 Root 권한을 유지하며 공격 흔적을 숨기는 악성 도구.

APT ​

  • 유형: 장기간 특정 조직을 표적으로 공격하는 지능형 지속 위협의 특징을 묻는 문제
  • 규칙: APT는 명확한 표적을 정해 장기간 정보를 수집하고 다양한 공격 기법으로 지속적으로 침투한다.
  • 함정: 단일 공격 기법이 아니라 제로데이, 악성코드, 사회공학 등을 복합적으로 사용하는 장기 공격이다.
  • 예시: 특정 기업이나 기관을 대상으로 내부 직원을 노린 피싱 후 침투, 정보 수집, 유출을 반복한다.
  • 메모: APT = Advanced Persistent Threat = 지능형 지속 위협.

프로그래밍 ​

Java static 필드와 싱글턴 객체 공유 ​

java
class Connection {

    private static Connection _inst = null;
    private int count = 0;

    static public Connection get() {
        if (_inst == null) {
            _inst = new Connection();
            return _inst;
        }
        return _inst;
    }

    public void count() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

public class main {

    public static void main(String[] args) {

        Connection conn1 = Connection.get();
        conn1.count();

        Connection conn2 = Connection.get();
        conn2.count();

        Connection conn3 = Connection.get();
        conn3.count();

        conn1.count();
        System.out.print(conn1.getCount());
    }
}
  • 유형: Java의 static 필드와 싱글턴 객체 공유를 확인하는 문제
  • 규칙: static 필드 _inst는 클래스 전체에서 하나만 존재하며 Connection.get()은 항상 같은 객체를 반환한다.
  • 함정: count는 static이 아니지만 conn1, conn2, conn3가 모두 같은 객체를 가리키므로 값이 함께 누적된다.
  • 예시: conn1.count(), conn2.count(), conn3.count(), conn1.count()가 같은 객체의 count를 총 4번 증가시킨다.
  • 메모: 단순 static 문제가 아니라 static 필드로 하나의 객체만 생성하는 싱글턴 패턴 문제이다.

Java 상속과 오버라이딩 ​

java
class classOne {
    int a, b;

    public classOne(int a, int b) {
        this.a = a;
        this.b = b;
    }

    public void print() {
        System.out.println(a + b);
    }
}

class classTwo extends classOne {
    int po = 3;

    public classTwo(int i) {
        super(i, i + 1);
    }

    @Override
    public void print() {
        System.out.println(po * po);
    }
}

public class main {
    public static void main(String[] args) {
        classOne one = new classTwo(10);
        one.print();
    }
}
  • 유형: Java 상속과 메서드 오버라이딩에 따른 동적 바인딩 문제
  • 규칙: 오버라이딩된 인스턴스 메서드는 참조 변수 타입이 아니라 실제 생성된 객체 타입의 메서드가 실행된다.
  • 함정: one의 선언 타입은 classOne이지만 실제 객체는 classTwo이므로 classTwo.print()가 호출된다.
  • 예시: classOne one = new classTwo(10);에서 one.print()는 classTwo의 print()를 실행한다.
  • 메모: 부모 생성자로 a = 10, b = 11이 저장되지만 부모의 print()가 호출되지 않아 21은 출력되지 않는다.

C 문자 판별과 시저 암호 변형 ​

c
#include <stdio.h>
#include <ctype.h>

int main() {
    char *p = "It is 8";
    char result[100];
    int i;

    for (i = 0; p[i] != '\0'; i++) {
        if (isupper(p[i]))
            result[i] = (p[i] - 'A' + 5) % 25 + 'A';
        else if (islower(p[i]))
            result[i] = (p[i] - 'a' + 10) % 26 + 'a';
        else if (isdigit(p[i]))
            result[i] = (p[i] - '0' + 3) % 10 + '0';
        else
            result[i] = p[i];
    }

    result[i] = '\0';
    printf("%s\n", result);

    return 0;
}
  • 유형: C의 문자 판별 함수와 문자 코드 연산을 이용해 문자열을 변환하는 문제
  • 규칙: 대문자는 5칸, 소문자는 10칸, 숫자는 3칸 이동시키고 공백 등은 그대로 저장한다.
  • 함정: 문자를 직접 이어 붙이는 것이 아니라 ASCII 코드 차이를 계산하며, 마지막에 \0을 넣어야 문자열이 완성된다.
  • 예시: 't'는 소문자이므로 (19 + 10) % 26 = 3이 되어 'd'로 변환된다.
  • 메모: isupper, islower, isdigit로 문자 종류를 구분하는 시저 암호 변형 문제이다.
