정보처리기사 실기 ​

  1. 기출 풀이 1시간
  2. 기출 오답 40분
  3. 키워드 암기 40분

한 달도 안 남았으면 목표는 **“실기 전체를 이해”가 아니라 “60점 이상을 안정적으로 만드는 것”**이어야 합니다. 정보처리기사 실기는 Q-Net 기준 2026년 출제기준이 별도로 적용되고, 정보처리기사 종목 상세에 2023~2025 출제기준과 2026 출제기준이 구분되어 있습니다. 따라서 지금은 최신 교재나 강의보다도 2026 기준 + 최근 기출 복원 + 프로그래밍/SQL 반복이 우선입니다.

1. 우선순위 ​

가장 먼저 잡아야 할 것은 다음 4개입니다.

  1. 프로그래밍 언어 활용
  2. SQL 응용
  3. 데이터 입출력 / DB
  4. 소프트웨어 개발 보안, 테스트, 네트워크, 운영체제 기초

실기에서 프로그래밍과 SQL은 암기로만 해결하기 어렵고, 반대로 제대로 잡으면 점수 회수가 큽니다. 최근 교재 설명에서도 2026 대비판은 프로그래밍 언어 쪽이 강화되었다고 설명합니다.

2. 남은 기간별 전략 ​

D-28 ~ D-21: 기출로 범위 파악 ​

이 시기에는 기본서 정독을 하면 안 됩니다.

최근 3~5개년 실기 기출 복원 문제를 먼저 풀면서 다음처럼 분류하세요.

분류표시
바로 맞힘O
알 것 같은데 틀림△
아예 모름X
외우면 되는 문제암기
코드/SQL 추적 문제연습

핵심은 “내가 뭘 모르는지”를 빠르게 보는 것입니다. 이때 정답률이 낮아도 괜찮습니다. 목적은 실력 측정이 아니라 출제 언어에 익숙해지는 것입니다.

D-20 ~ D-10: 점수 구간 만들기 ​

이 10일이 제일 중요합니다. 매일 아래 순서로 하세요.

오전 또는 첫 공부 시간에는 프로그래밍 1시간을 합니다. C, Java, Python 문법 중 특히 반복문, 배열, 문자열, 재귀, 포인터/주소 개념, 형변환, 출력 결과 예측 문제를 봐야 합니다.

그다음 SQL 40분을 합니다. SELECT, WHERE, GROUP BY, HAVING, ORDER BY, JOIN, 서브쿼리, 집계함수, DDL/DML 정도는 손으로 결과를 추적할 수 있어야 합니다.

마지막으로 암기 파트 1~1.5시간을 합니다. 보안 공격 기법, 테스트 종류, 결합도/응집도, 디자인 패턴, UML, 트랜잭션, 정규화, OS/네트워크 용어를 짧게 반복하세요.

이 기간에는 “외운다”보다 틀린 문제를 다시 맞히는 것이 더 중요합니다.

D-9 ~ D-4: 실전 회독 ​

이때는 하루에 최소 1회분씩 실전처럼 푸세요.

시간을 재고 풀고, 채점하고, 틀린 문제를 다음 네 가지로 나눕니다.

오답 원인처리
용어를 몰랐다단어장에 추가
코드 추적을 틀렸다같은 유형 3문제 추가
SQL 실행 순서를 몰랐다SELECT 처리 순서 복습
문제를 잘못 읽었다문제 조건 밑줄 습관

실기에서는 “아는 것 같은데 단어가 정확히 안 나오는 문제”가 많습니다. 그래서 답안을 쓸 때는 한글명, 영문 약어, 핵심 키워드를 같이 외우는 편이 좋습니다. 예를 들어 CSRF, SQL Injection, XSS, 정규화, 트랜잭션, ACID, 결합도, 응집도처럼요.

D-3 ~ 시험 전날: 새 내용 금지 ​

마지막 3일에는 새 강의를 보면 안 됩니다.

할 일은 세 가지입니다.

  1. 최근 기출 오답 다시 풀기
  2. 프로그래밍/SQL 출력 문제 매일 10~20개
  3. 암기장만 반복

특히 전날에는 보안, 테스트, DB, 네트워크 용어를 훑고, 코드는 짧은 문제만 보세요. 긴 문제를 새로 파면 불안만 커집니다.

3. 하루 공부 루틴 예시 ​

하루 4시간 기준이면 이렇게 잡는 게 좋습니다.

시간내용
60분프로그래밍 출력/빈칸 문제
50분SQL 문제
70분기출 1회분 또는 반 회분
40분오답 정리
20분암기 카드 반복

하루 2시간밖에 안 되면 이렇게 줄이세요.

시간내용
50분프로그래밍
40분SQL
30분기출 오답/암기

시간이 부족할수록 프로그래밍 + SQL + 최근 기출 오답만 남기고 나머지는 줄여야 합니다.

4. 공부 자료 선택 기준 ​

자료는 많이 쓰지 마세요. 한 달 미만이면 자료가 많을수록 망합니다.

추천 구조는 이렇습니다.

교재 또는 요약서 1개, 최근 기출 복원 모음 1개, 직접 만든 오답 노트 1개.

강의는 전체 강의 완강용으로 쓰면 비효율적입니다. 모르는 단원만 보는 용도로 쓰세요. 특히 C 포인터, Java 상속/오버라이딩, SQL JOIN, 정규화, 보안 공격 기법 정도만 골라 보는 것이 낫습니다.

6. 현실적인 목표 점수 전략 ​

가장 현실적인 전략은 이겁니다.

프로그래밍/SQL에서 가능한 한 맞히고, 암기 파트에서 부분점수를 긁어모으는 방식입니다.

프로그래밍과 SQL을 버리면 합격 안정성이 크게 떨어집니다. 반대로 암기 파트를 전부 완벽히 하려고 하면 시간이 부족합니다. 그래서 지금은 다음 순서가 맞습니다.

other
1순위: 프로그래밍 출력 문제
2순위: SQL
3순위: 보안 / 테스트 / DB / 네트워크 빈출 용어
4순위: 나머지 이론
